# Code Style

- Prefer clarity over cleverness; write code that reads like prose.
- Keep functions short and focused on a single responsibility.
- Use descriptive variable and function names; avoid abbreviations.
- Add type annotations to all function signatures (Python) or use TypeScript strict mode (JS/TS).
- Do not leave dead code or commented-out blocks in the codebase.
- Prefer early returns over deeply nested conditionals.
- Use consistent formatting; defer to the project's formatter (e.g. ruff, prettier).
